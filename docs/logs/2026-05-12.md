# 2026-05-12 开发日志

## 1. 新增 `--use_llm` 参数（`00d5101`、`3d456c4`）

**问题**：每次运行 `translate_type_rag.py` 都要调 LLM，即使只想用 `fixed_type_map` 做快速映射。

**方案**：
- `src/java/type_resolution/translate_type_rag.py` 新增 `--use_llm` 参数，默认 `true`
- `--use_llm=false` 时跳过 LLM 调用，仅用 fixed_type_map + 自定义类型
- `--use_rag` 仅在 `--use_llm=true` 时生效
- `scripts/java/translate_types.sh` 同步新增 `use_llm` 入参（第5个参数）

## 2. 自定义类型持久化（`8ee6eab`）

**问题**：`translate_type_rag.py` 和 `create_skeleton.py` 各自维护了一份 `get_custom_types`，每次都要扫描全部 schema 文件。

**方案**：
- 删除 `translate_type_rag.py` 中本地 `get_custom_types`，统一用 `src/java/utils/get_custom_types.py`
- 新增 `save_custom_types` 函数，结果持久化到 `data/java/type_resolution/<project>/custom_types.json`
- 每次运行都会更新，`create_skeleton.py` 后续可直接读 JSON

## 3. cjc 编译验证加入自定义类型桩（`c2c9969`）

**问题**：LLM 将 `ArrayList<Student>` 正确翻译后，`is_type_loadable` 编译验证时因找不到 `Student` 定义而误判为失败。

**方案**：在 cjc 测试程序中为所有自定义类型生成空类桩定义：
```cangjie
class Student {}
class StudentManager {}
```

嵌套类取简单名（仓颉不支持嵌套类），如 `Outer.Inner` → `class Inner {}`。
