# 使用说明

AnyHashable.cj直接放到x2cangjie文件夹下即可

### 第 1 步：激活 conda 环境

``` 
conda activate x2cangjie
```

### 第 2 步：创建 HashMap 测试 Java 项目

创建一个最小化的 Java 项目让流水线解析。以 `JavaFeatureTest` 为模板：

```
# 在容器内
cd /home/x2cangjie

# 创建测试 Java 源文件
mkdir -p projects/java/original_projects/HashMapTest/src/main/java/com/test

cat > projects/java/original_projects/HashMapTest/src/main/java/com/test/HashMapTest.java << 'EOF'
package com.test;

import java.util.HashMap;

public class HashMapTest {
    public void testHashMap() {
        HashMap<Object, String> map = new HashMap<>();
        map.put("key", "value");
        String v = map.get("key");
    }
}
EOF
```

* **作用：**生成测试文件`HashMapTest.java`
* **生成文件：**`projects/java/original_projects/HashMapTest/src/main/java/com/test/HashMapTest.java`

* **代替方法：**将`HashMapTest.java`直接放到对应位置

### 第 3 步：创建 cleaned_final_projects（跳过完整预处理）

`HashMapTest `是个简单测试项目，没有` pom.xml`，不能走 add_plugin → merge_jar → generate_cg → reduce 流程。直接手动创建` cleaned_final_projects `目录：

``` 
cd /home/x2cangjie

# 创建目标目录，把 original_projects 中的 HashMapTest 拷贝过去
mkdir -p projects/java/cleaned_final_projects
cp -r projects/java/original_projects/HashMapTest projects/java/cleaned_final_projects/HashMapTest
```

* **作用：**创建` cleaned_final_projects`，直接将`HashMapTest`复制到`cleaned_final_projects`中
* **生成文件：**`projects/java/cleaned_final_projects/HashMapTest`

### 第 4 步：运行流水线生成 Schema

```
# 创建 schema
cd /home/x2cangjie
export PYTHONPATH=/home/x2cangjie:$PYTHONPATH #根据项目实际位置修改

# 需要配置 model_configs.yaml (复制示例并填入 LLM API key)
cp configs/model_configs.yaml.example configs/model_configs.yaml
# 编辑 configs/model_configs.yaml 填入你的 API key

bash scripts/java/create_schema.sh HashMapTest gpt-4o-2024-11-20 0.0 ""
```

* **作用：**这会用` tree-sitter` 解析` projects/java/cleaned_final_projects/HashMapTest/ `下的 Java 文件，输出到` data/java/schemas/gpt-4o-2024-11-20/0.0/HashMapTest/`。

* **生成文件：**`data/java/schemas/gpt-4o-2024-11-20/0.0/HashMapTest/`

* **验证：**可以在对应文件夹下看到`HashMapTest.src.main.com.test.HashMapTest.json`文件

* **问题：**第9行bash，存在Windows 换行符（CRLF \r\n）问题。脚本在 Windows 上编辑/克隆，行尾带了 \r，bash 无法解析。

  可以：

  ```
  # 在容器内执行
  cd /home/x2cangjie
  sed -i 's/\r$//' scripts/java/*.sh
  # 然后重新运行：
  bash scripts/java/create_schema.sh HashMapTest gpt-4o-2024-11-20 0.0 ""
  ```

  或者改为使用：

  ```
  python src/java/decomposition/create_schema.py \
      --project_name=HashMapTest \
      --model=gpt-4o-2024-11-20 \
      --temperature=0.0 \
      --suffix=""
  ```

###  第 5 步：生成依赖顺序（可跳）

理论上可以不执行这一步，因为本测试只含有一个测试函数，


##### 生成 `traversal.json`

```
python utils.py --project_name=HashMapTest --function=parse_dependencies --suffix=
```

* **作用：**生成依赖顺序
* **生成文件：**`data\java\dependencies\HashMapTest`

### 第 6 步：类型翻译（需要 LLM API）(可跳)

==由于我们修改了静态类型映射表`fixed_type_map.json`，所以实际不会调用LLM进行类型翻译==

```
cd /home/x2cangjie
# 1. 配置 API key
cp configs/model_configs.yaml.example configs/model_configs.yaml

# 2. 运行类型翻译
bash scripts/java/translate_types.sh HashMapTest gpt-4o-2024-11-20 0.0 "" true
```

* **作用：**对` HashMap<Object, String>、HashMap<>、String`等类型逐一翻译
  - `void → Unit、String → String` 这类已在` fixed_type_map `中的直接命中，不走 LLM
  - 不在` fixed_type_map` 中的泛型类型（如 HashMap<Object, String>）才调 LLM
  - 每个翻译结果用 cjc 编译验证（需要仓颉 SDK 已安装）
  - 成功的翻译会写入` universal_type_map_final.json`

### 第 7 步：生成仓颉骨架

``` 
bash scripts/java/create_skeleton.sh HashMapTest gpt-4o-2024-11-20 "" 0.0
```

* **作用：**在 `data/java/skeletons/HashMapTest/ `下生成：
  - `src/HashMapTest.cj `— 骨架文件
  - `cjpm.toml` — 项目配置
* **生成文件：**`data/java/skeletons/HashMapTest`

* **验证：**观察`data/java/skeletons/HashMapTest/src/HashMapTest.cj`

### 第 8 步：复制 AnyHashable.cj 到骨架项目

```
mkdir -p data/java/skeletons/HashMapTest/src/temp_test/runtime
cp AnyHashable.cj data/java/skeletons/HashMapTest/src/temp_test/runtime/
```

* 手动复制过去也可以
* **作用：**为了在第9步中确认，是否能成功进行`cjpm build`
